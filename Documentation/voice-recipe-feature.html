Part 1 — The big picture
In plain English — what this feature does
A user opens the app, taps one button, and speaks like they're telling a friend how to make their favourite dish. They might say "Take 2 cups of rice, wash it, add 3 cups of water, cook for 10 minutes, then add salt." When they stop speaking, the app thinks for 5 seconds — and then a fully formatted recipe appears on screen, ready to review and publish. No typing. No forms. Just talking.
Why this is a big deal
The #1 reason home cooks don't share recipes online is that typing out a full recipe with ingredients and steps is time-consuming and tedious. Voice removes that barrier completely. A 2-minute recording can produce a recipe that would take 15 minutes to type. That's a 7x improvement in creator productivity.
Two technologies working together
This feature uses two separate AI services in sequence. First, OpenAI Whisper listens to the audio and converts it into raw text — like a very accurate transcription service. Then, Claude AI reads that raw text and organises it into a structured recipe. Each tool does what it's best at.
The analogy: Think of it like having a very smart kitchen assistant. You talk, they listen (Whisper), understand what you're cooking (Claude), and write it all down perfectly formatted. You just review their notes before sharing them.

Here is every step in the system, from the user tapping the microphone to the structured recipe appearing on screen.

🎙️
Step 1 — User's device
User taps the mic and speaks
The user taps the big orange microphone button. The app starts recording using the device's microphone. A waveform animation shows the voice is being captured. The user speaks naturally — no special format required. When done, they tap Stop.
Web: MediaRecorder API Mobile: expo-av User action

Step 2 — Upload
Audio file is sent to the server
The recorded audio file (in .webm or .m4a format) is sent from the user's device to the RecipeVerse backend. This happens the moment they tap Stop. A loading animation appears: "Processing your recipe..."
Multipart form upload Next.js API route

Step 3 — Speech to text (Whisper)
Whisper converts audio to raw text
The server sends the audio file to OpenAI's Whisper API. Whisper listens to the audio and converts it into a raw text transcript. This takes about 2–4 seconds for a 2-minute recording. The result is plain text — exactly what the user said, with no structure yet.
OpenAI Whisper API Server-side only
🤖
Step 4 — AI structuring (Claude)
Claude reads the transcript and structures the recipe
The raw transcript is sent to Claude AI with a carefully written prompt. Claude reads the text and extracts: title, ingredients (with quantities and units), numbered steps, cook time, servings, notes, and difficulty. It returns this as structured JSON data that the app can read.

✅
Step 5 — Review screen
The structured recipe appears for the user to review
The app receives the JSON and instantly populates every field in the recipe form. The user sees their full recipe structured and ready. Fields the AI was unsure about are highlighted in yellow. The user reviews, makes any corrections, adds a photo, and taps Publish.
Pre-filled form User reviews and edits
🚀
Step 6 — Publish
Recipe goes live on RecipeVerse
User taps Publish. The recipe is saved to the database and appears instantly on their profile and in the feeds of their followers. A celebration animation plays: "Your recipe is live! 🎉"

Part 4 — Speech to text: what Whisper does
What Whisper is and what it does in this feature
Whisper is an AI model made by OpenAI that is specifically designed to convert spoken audio into written text. It is extremely accurate, works in 50+ languages, and handles accents, background noise, and natural speech patterns very well. Think of it as a professional transcriptionist that works in milliseconds.

Whisper's exact job in RecipeVerse

What Whisper does
  • ✓ Converts audio file to raw text
  • ✓ Handles accents and regional dialects
  • ✓ Works in English, Spanish, French, Arabic and 50+ more
  • ✓ Handles background noise reasonably well
  • ✓ Processes a 3-min recording in ~3 seconds
What Whisper does NOT do
  • ✗ It does NOT understand what the words mean
  • ✗ It does NOT organise text into ingredients/steps
  • ✗ It may struggle with very uncommon ingredient names
  • ✗ It does NOT work in real-time (file must be complete)
  • ✗ It does NOT handle very heavy background music well
Whisper's output example: For the input "Take 2 cups of rice, wash it, add 3 cups of water", Whisper returns exactly: "Take 2 cups of rice, wash it, add 3 cups of water." — just plain text. It doesn't know that "2 cups of rice" is an ingredient. That's Claude's job next.
Part 5 — AI structuring: what Claude does
What Claude does with the raw transcript
Claude receives the plain text from Whisper and a very specific set of instructions from us (called a "prompt"). Claude reads the text intelligently — like a smart human reading a recipe description — and organises everything into the right fields. It understands context, can infer a title from the description, and knows that "add salt" is a step, not an ingredient list item.

The AI prompt we send to Claude

The "prompt" is the instruction we give Claude. This is the most important part of the whole feature — a better prompt gives better results. Here is the actual prompt RecipeVerse should use:

System prompt sent to Claude API
You are a recipe structuring assistant for a cooking platform.
You will receive a raw voice transcript of someone describing a recipe.
Your job is to extract and structure it into clean JSON.

Rules:
- Extract only what the user explicitly said. Never invent information.
- If a field is not mentioned, set it to null — do not guess.
- For the title: make a short, descriptive guess based on the ingredients/method.
Mark it as "ai_generated": true so the UI can flag it for review.
- Quantities and units must be separated: "2 cups" → quantity: "2", unit: "cups"
- Steps must be numbered, one action per step.
- If a quantity is unclear (e.g. "some salt") set confidence: "low".

Return ONLY valid JSON in this exact format, nothing else:
{
  "title": "...",
  "title_ai_generated": true,
  "ingredients": [
    { "name": "rice", "quantity": "2", "unit": "cups", "confidence": "high" }
  ],
  "steps": [ { "number": 1, "instruction": "Wash the rice." } ],
  "cook_time_mins": 10,
  "servings": null,
  "notes": null,
  "overall_confidence": "high"
}
Why the prompt matters so much: The same transcript sent to Claude with a vague prompt vs. a specific prompt gives completely different results. The prompt above forces Claude to be honest about uncertainty, never invent data, and return a consistent format the app can always parse reliably.

Part 8 — MVP version vs advanced version

Build the MVP version first and get it in users' hands. Add the advanced features only after you have real feedback.

MVP version
Build in Week 4 · Ship in Week 8
  • Single tap to start/stop recording
  • Audio sent to Whisper for transcription
  • Transcript sent to Claude for structuring
  • All 6 recipe fields extracted (title, ingredients, steps, time, servings, notes)
  • AI-generated fields flagged in yellow
  • Full review screen before publishing
  • Retry if something goes wrong
  • "Type instead" fallback always available
  • Max recording length: 5 minutes
  • English language only at launch
  • Cost: ~$0.01 per recipe upload
Advanced version
Phase 2 · After real user feedback
  • Real-time transcription shown as user speaks
  • Multi-language support (50+ languages)
  • AI improves based on user corrections over time
  • Auto-detect language from audio
  • "Re-record just the ingredients" partial re-record
  • AI suggests a cover photo based on the recipe name
  • AI suggests relevant tags automatically
  • Cooking timer extracted from steps (e.g. "cook 10 min" → adds timer)
  • Nutritional info estimated from ingredients by AI
  • Voice editing: "Change step 2 to..." updates just that step
  • Background noise filtering before sending to Whisper
The most important Phase 2 addition: Real-time transcription (showing words appear on screen as the user speaks) dramatically increases trust. Users feel confident the app is "hearing" them correctly and can stop if something goes wrong. Build this in Phase 2 once MVP is validated.
