RecipeVerse Tech Stack

Every major technology category compared side by side — with one clear winner per category, explained in plain English. Budget-first, scalable later.

8 categories 3 options each MVP: ~$10–30/mo Scales to 100k+ users
01 — Frontend framework · what users see and interact with
🖥️
Frontend framework
Builds every screen — feed, recipe detail, profiles, create flow
Next.js 14 Free
  • Industry standard — most used in 2026
  • Built-in SEO so recipes rank on Google
  • Works as frontend AND backend in one
  • One-click deploy to Vercel
  • Massive community + component libraries
  • App Router has a learning curve
SvelteKit Free
  • Faster performance than React
  • Cleaner, simpler syntax
  • Smaller bundle sizes
  • Much smaller community
  • Fewer UI libraries available
  • Harder to hire developers
React + Vite Free
  • Fast dev server
  • Simple setup
  • Full React ecosystem
  • No SEO out of the box
  • No server-side rendering
  • Recipes won't rank on Google
🏆 Winner: Next.js 14 — SEO for recipe pages + full-stack in one codebase + massive ecosystem
02 — Backend / API · handles all logic, data, and AI calls
⚙️
Backend / API layer
Saves recipes, handles auth, calls AI, manages social actions
tRPC v11 Free
  • End-to-end type safety — bugs caught early
  • Lives inside Next.js — one codebase
  • No need to write API docs
  • Frontend + backend share types automatically
  • TypeScript only (not plain JS)
Next.js API Routes Free
  • Built into Next.js — zero setup
  • Familiar REST pattern
  • Can become public API later
  • No type safety between layers
  • More boilerplate to write
  • Easy to introduce bugs
Separate Node/Express Free
  • Full flexibility
  • Can scale independently
  • Two codebases to manage
  • Weeks of extra setup
  • Overkill for MVP
🏆 Winner: tRPC v11 inside Next.js — type safety + one codebase + fastest to build
03 — Database · stores all users, recipes, comments, likes, follows
🗄️
Database
Permanent storage for all platform data
Supabase Free tier
  • Free: 500MB DB + real-time subscriptions
  • PostgreSQL — most powerful open-source DB
  • Built-in auth, storage, and dashboard
  • Real-time feed updates included
  • Free tier pauses after 1 week inactivity
PlanetScale ~$29/mo
  • Excellent scaling
  • Zero-downtime schema changes
  • Great developer experience
  • No free tier anymore
  • MySQL lacks PostgreSQL features
  • Costs money from day one
Firebase Firestore Free tier
  • Real-time by default
  • No SQL needed
  • Google-backed reliability
  • NoSQL — bad for social relationships
  • Complex queries get expensive
  • Hard to migrate later
🏆 Winner: Supabase (PostgreSQL) + Drizzle ORM — free tier + real-time + best relational DB for social apps
04 — Authentication · signup, login, Google OAuth, sessions
🔐
Authentication
Handles all login flows securely — don't build this yourself
Clerk Free up to 10k MAU
  • Beautiful pre-built login UI
  • Google, Apple, email, phone all included
  • Perfect Next.js App Router integration
  • User dashboard included
  • Free for first 10,000 monthly users
  • Paid after 10k users (~$25/mo per 1k)
NextAuth.js Free / open-source
  • Free forever
  • Deep Next.js integration
  • Full control over flow
  • More setup required
  • You build all login screens
  • Auth v5 still has rough edges
Supabase Auth Free (included)
  • Already included with Supabase
  • Tight DB integration
  • Email + Google supported
  • Less polished than Clerk
  • More manual setup for social login
🏆 Winner: Clerk — best developer experience + free for 10k users + zero auth bugs
05 — Media storage · photos and videos uploaded by creators
📸
Media storage
Stores and serves all creator photos and recipe videos
Uploadthing Free tier
  • Built for Next.js — 20-minute setup
  • Free: 2GB storage + 10GB bandwidth
  • Handles large video uploads reliably
  • Type-safe file uploads
  • Fewer image transforms than Cloudinary
  • Newer — less proven at huge scale
Cloudinary Free tier
  • Industry standard — used by millions
  • Powerful image/video transforms
  • Free: 25GB storage + 25GB bandwidth
  • Auto WebP optimisation
  • More complex setup
  • Paid plans expensive with heavy video
AWS S3 + CloudFront Pay-as-you-go
  • Infinitely scalable
  • Very cheap at scale
  • Full control
  • Hours of complex setup
  • IAM permissions are a security trap
  • Way overkill for MVP
🏆 Winner: Uploadthing → upgrade to Cloudinary later — fastest setup + free tier + built for Next.js
06 — Search · find recipes by title, ingredient, cuisine, creator
🔍
Search
Powers recipe and creator search across the platform
PostgreSQL Full-Text Free (built-in)
  • Already in your Supabase DB — zero cost
  • Handles title, ingredient, and tag search
  • Good enough for 500k+ recipes
  • No extra service to manage
  • No typo tolerance ("chicen" ≠ "chicken")
  • Less smart relevance ranking
Algolia Free up to 10k records
  • Best-in-class search quality
  • Typo-tolerant + instant results
  • Used by Airbnb, Stripe
  • Gets expensive after free tier
  • Extra service to sync and manage
  • Overkill for MVP
Typesense Free self-hosted
  • Open-source Algolia alternative
  • Typo-tolerant and fast
  • Cheaper at scale
  • Extra DevOps to manage
  • Smaller community
  • Added complexity for MVP
🏆 Winner: PostgreSQL Full-Text now → Algolia when you hit 100k+ recipes — free + already in your stack + no extra service
07 — AI & speech-to-text · the voice-to-recipe feature
🎙️
Speech-to-text (voice → raw transcript)
Converts the user's audio recording into plain text
OpenAI Whisper $0.006/min
  • Most accurate available in 2026
  • Handles all accents + 50+ languages
  • 3-min recipe costs ~$0.018
  • Simple single-request API
  • Small per-minute cost
  • No real-time streaming (file must be complete)
Google Speech-to-Text 60 min free/mo
  • Free first 60 min/month
  • Real-time streaming available
  • 125+ languages
  • Slightly less accurate than Whisper
  • More complex setup
  • Pricing gets complex at scale
Deepgram $200 free credit
  • Very fast, low latency
  • Real-time available
  • $200 free to start
  • Smaller community
  • Less accurate on food terminology
🏆 Winner (speech): OpenAI Whisper — most accurate + handles all accents + pennies per recipe
🧠
AI structuring (transcript → recipe fields)
Reads raw text and extracts title, ingredients, steps, cook time, servings
Claude API ~$0.003/recipe
  • Best at following precise JSON instructions
  • Most reliable structured output
  • Handles long recipes easily
  • claude-sonnet-4-6 is fast + affordable
  • Small per-use cost
GPT-4o mini ~$0.002/recipe
  • Very cheap
  • Good JSON mode
  • Fast response
  • Same OpenAI account as Whisper
  • Slightly less precise than Claude
  • Consolidating both Whisper + GPT on one bill
Gemini Flash Free tier
  • Free up to generous limits
  • Fast inference
  • Can handle audio natively (future)
  • Less reliable structured JSON output
  • API less mature
  • Inconsistent recipe extraction
🏆 Winner (AI): Claude API (claude-sonnet-4-6) — best structured output + most reliable JSON + great instruction-following
08 — Hosting & deployment · where the app lives on the internet
☁️
Hosting & deployment
Push code → app goes live. Handles traffic, SSL, global CDN.
Vercel Free hobby tier
  • Made by the Next.js team — perfect match
  • Push to GitHub → live in 30 seconds
  • Free SSL, CDN, analytics included
  • Preview URLs for every PR
  • Zero server management
  • Pro plan needed for teams ($20/mo)
  • Long-running functions have limits on free
Railway ~$5/mo
  • Good for non-Next.js backends
  • Simple pricing
  • Docker support
  • Not optimised for Next.js
  • Costs from day one
  • Less global CDN
AWS / GCP / Azure Complex
  • Infinitely scalable
  • Full infrastructure control
  • Industry standard at scale
  • Weeks of configuration
  • Requires DevOps expertise
  • Easy to accidentally overspend
  • Total overkill for startup MVP
🏆 Winner: Vercel — made for Next.js + zero config + free tier + global edge network
09 — The recommended MVP stack · one table, everything decided

RecipeVerse — complete recommended stack

Every tool chosen for budget, speed to build, and clear upgrade path

Frontend
Next.js 14 + React 19 + Tailwind CSS v4 Free
Builds all screens. Recipe pages rank on Google (free organic traffic). Tailwind makes UI fast to build. React 19 is the most hireable skill on earth.
Backend
tRPC v11 inside Next.js Free
Same codebase as frontend. Type-safe from database to UI — bugs caught before users see them. No REST boilerplate to write.
Database
Supabase (PostgreSQL) + Drizzle ORM Free tier
Free PostgreSQL database with real-time subscriptions for the live feed. Drizzle writes queries in TypeScript. Dashboard to view all data visually. Free up to 500MB.
Auth
Clerk Free up to 10k users
Google login, email signup, password reset — all built in. Pre-built UI. Saves 2–3 weeks of development. Free for the first 10,000 monthly active users.
Media
Uploadthing Free tier
Built for Next.js. Set up in 20 minutes. Handles photo and video reliably. Free: 2GB storage + 10GB bandwidth. Upgrade to Cloudinary when you need advanced image transforms.
Search
PostgreSQL Full-Text Search Free (built-in)
Already in Supabase — zero extra cost. Searches recipe titles, ingredients, and tags. Handles up to 500k+ recipes. Upgrade to Algolia only when search quality becomes a complaint.
Voice AI
Whisper API + Claude API ~$0.022 / recipe
Whisper transcribes the audio ($0.018). Claude structures it into recipe fields ($0.004). Total: 2.2 cents per voice upload. At 1,000 uploads/month = $22. Essentially free at MVP scale.
Hosting
Vercel Free tier
Push to GitHub and it's live. Free SSL, global CDN, preview URLs. Zero server management. Upgrade to Pro ($20/mo) only when you add team members.
Mobile (Phase 2)
React Native + Expo SDK 52 Free
Do NOT build this for MVP. A mobile-responsive web app covers early users. Add native iOS + Android in Phase 2 once the web product is validated.

~$10–30/mo

Vercel Pro ($20/mo) Supabase Pro ($25/mo) Clerk scales with users Upgrade to Cloudinary Add Algolia for search Launch mobile app (Expo)

Dedicated Postgres instance Redis for feed caching CDN for video delivery Self-host Typesense Background jobs (Inngest) Real-time with Supabase channels
~$1k–5k/mo
The key principle: Every tool has a clear upgrade path. You never hit a wall where you throw everything away. Supabase → dedicated Postgres. Uploadthing → Cloudinary. Postgres search → Algolia. Upgrade each piece only when you actually need to — not before.
